- 監査やチェックを「仕組み化」する土台を用意する。

### 施策
| 施策 | 内容 | 状態 |
|------|------|------|
| ドキュメントの現状同期 | `docs/SITE_STRUCTURE_AND_CONTENT.md` の記事一覧・カテゴリ・セクション構成を `lib/articles.ts` と一致させる。家計管理・改善効果の試算を「準備中」から実カテゴリに更新。 | 実施 |

## フェーズ9: 内部リンク・構造化データの強化

### 目的
- 内部リンクのリンク切れを防ぎ、クロール効率とユーザー体験を維持する。
- ピラー・クラスターの関係を明示し、関連記事の導線を最適化する。

### 施策
| 施策 | 内容 | 状態 |
|------|------|------|
| 内部リンクのビルド時チェック | `lib/articles.ts` の `relatedLinks` および記事本文内の `/articles/:slug` が、存在する slug を指しているか検証するスクリプトを用意し、`npm run build` 前や CI で実行する。 | 実施 |
| ピラー・クラスターの明示 | 各カテゴリの「親記事」（order: 0 や badge: おすすめ）と子記事の対応をドキュメント化し、関連リンクの追加・見直し時に参照する。 | ドキュメント化 |
| 構造化データの見直し | 既存の Article / FAQPage / BreadcrumbList に加え、必要に応じて HowTo や QAPage を検討する（優先度は中）。 | 検討 |

## フェーズ10: パフォーマンス・計測と継続改善

### 目的
- 表示速度と Core Web Vitals を維持・改善する。
- 検索・クリック・離脱の計測から「次の一手」を決めるサイクルを回す。
- 記事の更新サイクルと `dateModified` の運用を確立する。

### 施策
| 施策 | 内容 | 状態 |
|------|------|------|
| Core Web Vitals の監視 | Vercel Analytics や PageSpeed Insights で LCP・CLS・FID を定期確認。Recharts のクライアント描画が LCP に影響しないか注意する。 | 継続 |
| 計測の見直し | Search Console の表示クエリ・順位・クリック率を確認し、10位前後のクエリを強化する。GA4 の導線別クリック・シミュ離脱率を把握する。 | 継続 |
| 記事更新と dateModified | 記事を更新したら、該当記事の JSON-LD の `dateModified` を更新する。将来的に `articlesData` に `dateModified` を持たせ、サイトマップに反映する。 | 運用・任意 |
| 免責・注意書きの統一 | 金融コンテンツとして、各ページで「概算・参考」「契約優先」の旨を必要に応じて明記する。 | 随時 |
