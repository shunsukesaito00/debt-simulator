# 読みやすさのベンチマーク（観察メモ）

個人サイト・金融リテラシー系サイトの**レイアウトと情報設計**を参考にし、**文章の転載は行わない**。構成・余白・見出しの切り方だけをメモする。

## 参照URL（追記用）

運営者が「読みやすい」と感じたサイトを、以下に **URL と観察メモ** で追記する。

| サイト名・メモ | URL | 真似したい点（チェック） |
|----------------|-----|-------------------------|
| （例）生活困窮者等向け相談（独立行政法人） | https://www.cfa.go.jp/policies/kokumin_seikatsu | 公的リンクの置き方・注意書きのトーン |
| （例）消費者庁（消費者の皆様へ） | https://www.caa.go.jp/ | 箇条書き・階層のはっきりした見出し |
| HAIAGARU 等・個人ブログ | （未記入） | 行間・コールアウト・目次・一人称の距離感 |

> **HAIAGARU** など名称だけ分かっている場合は、正確な URL をここに貼ってから観察項目を埋める。

## 観察チェックリスト（ページを開いて確認）

- [ ] **ファーストビュー**: タイトル直下で「何が分かるページか」が1〜2文で言えるか
- [ ] **行間・字サイズ**: 本文が `text-sm` だけで詰まっていないか（`text-base` + 段落間）
- [ ] **見出し密度**: 長い段落の途中に `h2`/`h3` があるか（スキャンできるか）
- [ ] **リスト**: 手順・比較は箇条書きか
- [ ] **注意・補足**: 枠・引用・ラベルで区切られているか
- [ ] **体験の扱い**: 数値記事は「例の人」のシナリオがあるか／日記記事は時系列か
- [ ] **外部情報**: 公的サイトへのリンクがあれば、**引用ではなくリンク＋要約**になっているか
